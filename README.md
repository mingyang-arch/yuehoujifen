# 阅后即焚 (One-Time Secret Sharing)

一个极简的一次性秘密分享网站。用户可以输入秘密内容，生成一个只能访问一次的链接。一旦内容被读取，服务器立即销毁数据。

## 功能特点

- 🔒 一次性访问：链接只能打开一次
- 🔥 阅后即焚：内容读取后立即销毁
- ⏰ 自动过期：未读取的秘密24小时后自动清理
- 🎨 极简设计：清爽的界面，专注核心功能
- 📱 响应式布局：支持移动端和桌面端

## 快速开始

### 安装依赖

```bash
npm install
```

### 启动服务器

```bash
# 生产模式
npm start

# 开发模式（自动重启）
npm run dev
```

服务器将在 `http://localhost:3000` 启动。

## 使用方法

1. 访问首页
2. 在文本框中输入秘密内容
3. 点击"生成一次性链接"
4. 复制生成的链接并分享给接收者
5. 接收者打开链接后，内容将被显示并立即销毁

## 技术架构

- **后端**: Node.js + Express
- **存储**: 内存存储（Map）
- **前端**: 原生 HTML/CSS/JavaScript
- **ID生成**: nanoid

## 安全说明

- 当前版本使用内存存储，服务器重启后数据会丢失
- 生产环境建议使用 Redis 等持久化存储
- 建议配置 HTTPS 确保传输安全
- 未实现用户认证，任何人都可以创建秘密

## 生产部署建议

1. 使用 Redis 替代内存存储
2. 配置 HTTPS
3. 添加速率限制（防止滥用）
4. 设置内容长度限制
5. 添加日志记录
6. 配置环境变量
